# 各偉人ページに「逸話・エピソード」セクション追加 設計

- 作成日: 2026-05-24
- 目的: 各偉人ページに人物像が立体的に立ち上がる小さな逸話を 1-3 個追加し、コンテンツの厚みを増す
- 経緯: 「縦深化(深める)」軸の継続施策。relatedPeople(関係性)に続いて「逸話」を追加することで、人物の「外側の繋がり」+「内側のキャラ」両面が見える状態を作る

## スコープ

### 含む (v1)
- 全 90 偉人ページの本文末尾に `## 逸話・エピソード` セクションを追加
- 各偉人につき 1-3 個の逸話を `### 逸話タイトル` + 段落 で記述
- 既に本文に書かれている内容との重複は避ける
- 新規キュレーションは AI(並列 agent)が実施、Wikipedia 本人記事冒頭を主ソース

### 含まない (今回スコープ外)
- 構造化データ(frontmatter `episodes:`)の導入 → 本文 markdown に直接書く
- 専用 component / カード型表示 → 既存の body markdown レンダリングをそのまま利用
- 検索・フィルタ機能 → 単なる読み物として配置

## 動機

各偉人ページの本文は伝記的・歴史的内容が中心で、その人物の「人となり」や「印象的な一場面」を伝える小さな逸話が散在している程度。

例: 犬養毅の「話せばわかる」「問答無用」、西竹一(バロン西)のロサンゼルス五輪逸話、御木本幸吉の「世界中の女性の首を真珠で飾りたい」などは、本文中に断片的に登場するか、まったく登場しない場合がある。

これらを「逸話・エピソード」セクションとして集約・追加することで、読み物としての魅力と人物像の立体感を大きく増す。

## 設計

### 配置

各偉人 md の本文末尾、`## 青山霊園に眠る` 章の直前(なければ本文の一番末尾)に挿入:

```markdown
## 逸話・エピソード

### <短いキャッチー見出し>

<本文 1-3 段落、150-400 字程度>

### <別の逸話の見出し>

<本文>
```

- 1 人あたり **1-3 個**(目安)
- 「面白い逸話が無い・既出と重複する」場合は **0 個でも OK**
- 既に本文で語られているエピソードは再記述しない(新切り口だけ追加)

### 逸話の選び方

人物像が立体的に立ち上がる小さな話を選ぶ:

1. **名言・印象的な発言**: 「話せばわかる」「ボッコちゃん」発想の経緯 etc.
2. **趣味・人柄**: ゴルフ、書道、絵画、執筆、酒など
3. **家族・友人との会話**: 子供・配偶者・盟友との交流
4. **最期の言葉・場面**: 暗殺前後、病床の言葉、死の場面
5. **小さな事件・遭遇**: 留学先での出来事、有名人との初対面、トラブル
6. **物語性のある裏話**: 自身の作品・業績の誕生背景

### 重複回避ルール

各 agent は対象 md の本文を全部読んでから新規追加。既に書かれている話題は転記しない。例:
- 大久保利通: 紀尾井坂の襲撃 → 本文済 → 追加しない
- 大久保利通: 倹約家として知られた話 → 本文未出 → 追加可能

### Fact-check

- Wikipedia 本人記事冒頭が主ソース
- WebFetch が使えない場合は md 本文 + 既存知識のみで OK
- 確証のない逸話・年代は書かない
- 出典が必要な場合は `references` への追加は v2 以降の判断(まずは shortな逸話のみ)

### マークダウン記法ルール(プロジェクト規約)

- **本文中に `**bold**` 記法を使わない**(commit `942a704` の 3,500 個 strip 経緯)
- 強調は文脈と見出しで表現
- セクション見出しは `### 〇〇`(`## 逸話・エピソード` の下に並ぶ)

## 実装方針

### 工程

1. **設計レビュー**(本ドキュメント) → commit
2. **6 並列 agent 分担**: 90 名 ÷ 6 = 15 名/グループ
3. 各 agent:
   - 担当 15 名の md を Read → 本文を把握
   - 既出内容と重複しない新規逸話を 1-3 個キュレーション
   - Edit で本文末尾(`## 青山霊園に眠る` の直前)にセクション追加
4. **build 検証**: `npm run build` で 0 エラー(YAML 検証 + markdown 検証)
5. **目視確認**: `npm run dev` で代表偉人ページの表示崩れチェック
6. **commit & push**

### 副作用・リスク

- **既存本文の改変リスク**: agent は本文末尾に追加のみ、既存段落の改変は禁止
- **キュレーション偏り**: AI 知識の偏りで「教科書的に有名な逸話」に偏る可能性。許容範囲とする
- **重複混入**: 本文ですでに語られた話を再記述してしまう可能性。各 agent には「本文を全部読んでから追加」と厳命
- **bold 記法**: プロジェクトルール厳守

## 完了基準

- 全 90 偉人ページに `## 逸話・エピソード` セクションが検討された(逸話が無い人物は 0 個で空セクションも追加しない選択肢あり)
- `npm run build` が 0 エラー
- 代表 5 ページの目視確認で表示崩れなし
- 本文との重複が目視で問題なし
