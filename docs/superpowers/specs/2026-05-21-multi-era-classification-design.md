# 多元号モデルへの再分類

- 作成日: 2026-05-21
- 関連: `docs/superpowers/specs/2026-05-21-top-page-grouping-and-filter-design.md`(時代フィルタの基盤)

## 背景

トップページの時代フィルタを動作確認した結果、`era` が単一元号(没年元号を暗黙ルールで採用)になっており、長い活動期を持つ人物で違和感があった(例: 松方正義の era=大正、北里柴三郎の era=昭和、犬養毅の era=昭和)。「その人物を表現するにあたって適切な元号」を返すよう、データモデルとルールを見直す。

## ゴール / 非ゴール

**ゴール**
- 1 人物に 1〜2 個の元号を持たせられるデータモデルへ
- 判定ルールを明文化(主活動期 = 代表業績・最高職位の元号)
- 全 29 名を新ルールで再分類
- トップページの時代フィルタは OR 条件(該当元号のどれか 1 つに合致すれば表示)
- カードのバッジは「明治・大正」のような中黒区切り
- 個別ページ(`/people/<slug>/`)の「時代」表示も同形式

**非ゴール**
- 3 元号以上の人物を扱う(犬養毅・後藤新平など三元号活動者でも max 2)
- 元号の濃淡(主・副)を構造化する(判断揺れを増やすので持たない)
- 時代範囲外の元号(平成・令和)対応

## データモデル

### スキーマ変更

`src/content.config.ts`:

```ts
// before
era: z.enum(['江戸', '明治', '大正', '昭和']),

// after
era: z.array(z.enum(['江戸', '明治', '大正', '昭和'])).min(1).max(2),
```

### Markdown frontmatter

```yaml
# before
era: 明治

# after(配列フォーマット、1 個でも配列で書く)
era:
  - 明治
```

または:

```yaml
era: [明治, 大正]
```

## 判定ルール(明文化)

1. **代表業績・最高職位が属する元号を 1 つ採用するのを原則とする**
   - 政治家: 首相・大臣など最高職位の在任時期
   - 軍人: 主要戦役・最高階級の時期
   - 文学者: 代表作の発表時期
   - 学者: 主要業績の発表時期
   - 実業家: 創業・代表事業の確立時期
2. **代表業績が明確に 2 元号にまたがる場合のみ、両方を書く(max 2)**
   - 例: 後藤新平(台湾統治・満鉄=明治、東京市政・震災復興=大正)→ 明治・大正
   - 例: 浜口雄幸(蔵相・内相=大正末、首相暗殺=昭和初期)→ 大正・昭和
3. **没年元号は判定要素にしない**
   - 単に晩年まで生きただけで活動が薄い元号は含めない
4. **3 元号にまたがる長寿活動家も、代表業績で 2 つに絞る**
   - 例: 犬養毅(明治の文相経験あり、大正の護憲、昭和の首相暗殺)→ 大正・昭和(明治の文相は代表業績ではないため除外)
5. **判断揺れが大きい場合、シンプルな方を採る**(2 つに迷ったら 1 つ)

## UI / 実装変更

### 1. データ属性

```astro
<!-- before -->
<article class="person-card" data-era={person.data.era} ...>

<!-- after -->
<article class="person-card" data-eras={person.data.era.join(',')} ...>
```

### 2. フィルタ判定ロジック

```ts
// before
const matchEra = currentEra === 'all' || article.dataset.era === currentEra;

// after
const matchEra =
  currentEra === 'all' ||
  (article.dataset.eras ?? '').split(',').includes(currentEra);
```

### 3. presentEras 集計

```ts
// before
const presentEras = ERA_ORDER.filter((era) => people.some((p) => p.data.era === era));

// after
const presentEras = ERA_ORDER.filter((era) => people.some((p) => p.data.era.includes(era)));
```

### 4. バッジ表示

```astro
<!-- トップカード -->
<span class="era-badge">{person.data.era.join('・')}</span>

<!-- 個別ページ -->
<tr><th>時代</th><td>{person.data.era.join('・')}</td></tr>
```

### 5. noscript fallback

変更なし(`.era-filter` 全体を隠す既存実装で対応)。

## 受け入れ基準

1. `npm run build` が通る(zod 配列スキーマで全 29 .md が validation を通過)
2. トップで「明治」チップ選択時、20 名(明治を含む人物)が表示される
3. 「大正」チップで 11 名、「昭和」チップで 7 名、「江戸」チップで 1 名
4. 「江戸」+「政治家」など 0 件の組合せで該当セクションが消える
5. カードバッジに「明治・大正」のような複数元号が中黒区切りで表示される
6. 個別ページの「時代」欄も同形式
7. URL hash `#era=明治&cat=政治家` から復元が動く
8. JS 無効でも全件表示される(noscript fallback)
